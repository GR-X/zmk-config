# 如何查代理端口并让 Git 使用

## 1. 先试本机已监听的端口 17890

当前你电脑上 **17890** 在监听，可能是代理。在 PowerShell 里执行：

```powershell
git config --global http.proxy http://127.0.0.1:17890
git config --global https.proxy http://127.0.0.1:17890
cd C:\Users\96447\Desktop\zmk_program\thinkpad
git push
```

若 push 成功，说明代理端口就是 17890。

---

## 2. 在代理软件里查端口（推荐）

| 软件 | 哪里看端口 |
|------|------------|
| **Clash** | 打开主界面 → 设置 / Settings → 端口 (Port)，或看「混合端口 / Mixed Port」；常见 7890、7891、7897 |
| **Clash Verge / Clash for Windows** | 设置里「端口」或「Port」，或点「复制终端代理命令」看端口号 |
| **V2RayN** | 设置 → 参数设置 → 本地监听端口，常见 10808、1080 |
| **其他** | 一般在「设置 / 参数 / 系统代理」里会有「HTTP 代理端口」或「Socks 端口」 |

查到端口后（例如 7890），执行：

```powershell
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

把 `7890` 换成你看到的端口。

---

## 3. 确认代理已开启

- 软件要**已启动**（托盘或任务栏有图标）
- 若用了「系统代理」或「TUN 模式」，确保已开启
- 用浏览器能打开 https://github.com 时，再试 `git push`

---

## 4. 取消代理（直连时）

```powershell
git config --global --unset http.proxy
git config --global --unset https.proxy
```
