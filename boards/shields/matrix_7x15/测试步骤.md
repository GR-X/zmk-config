# Matrix 7x15 键盘测试步骤

## 一、确认固件和连接

1. **烧录的必须是 matrix_7x15 的固件**  
   - 从 GitHub Actions 下载的 Artifacts 里，选 **matrix_7x15-nice_nano-zmk.uf2**（或名字里带 matrix_7x15 的），不要用 corne 的。
2. **用 USB 测试（先别用蓝牙）**  
   - 键盘插 USB 线连电脑，打开 **记事本**，光标点在记事本里。

---

## 二、一次只做一件事

### 步骤 1：看电脑是否识别键盘

- 插上 USB 后，在 **设备管理器** 里看是否多出「键盘」或「HID 设备」。
- 若完全没有新设备，或带黄叹号，先解决识别/驱动，再往下。

### 步骤 2：用「全 A」keymap 测按键

当前 keymap 已改成：**每个键都输出字母 A**。

1. 推送代码，让 GitHub Actions 重新构建，或本地用 west 构建 matrix_7x15。
2. 烧录新生成的 **matrix_7x15** 的 **.uf2**。
3. 键盘插 USB，打开记事本，**随便按键盘上任意一个键**（多按几个不同位置）。
4. 看记事本里是否出现字母 **A**。
   - **有 A**：说明至少有按键被识别，接下来再调键位/极性。
   - **完全没有 A**：继续步骤 3。

### 步骤 3：试「col 反相」配置（col 空闲为高时用）

若步骤 2 仍没有任何反应，且你硬件是 **col 有上拉、空闲时 col 为高**，可以试 col 反相：

- 打开 **matrix_7x15.overlay**，把 `&kscan0` 里 **row-gpios** 全部改成 **GPIO_ACTIVE_LOW**，**col-gpios** 全部改成 **(GPIO_ACTIVE_LOW | GPIO_PULL_UP)**。
- 保存后重新构建、烧录，再重复步骤 2（全 A keymap 不变，继续按任意键看是否出 A）。

---

## 三、结果怎么判断

| 现象 | 说明 |
|------|------|
| 按任意键都出 A | 矩阵和极性都对，把 keymap 改回正常键位即可。 |
| 按某些键出一串 A 或乱跳 | 可能是 row/col 顺序或极性不对，再对一下 overlay 和接线。 |
| 仍然完全没反应 | 检查：烧录的是不是 matrix_7x15 的 UF2、USB 是否被识别、是否试过 col 反相。 |

---

## 四、col 反相时 overlay 里要改的两处

在 **matrix_7x15.overlay** 里：

**row-gpios**：把 `GPIO_ACTIVE_HIGH` 改成 `GPIO_ACTIVE_LOW`（7 个都改）。

**col-gpios**：把 `(GPIO_ACTIVE_HIGH | GPIO_PULL_DOWN)` 改成 `(GPIO_ACTIVE_LOW | GPIO_PULL_UP)`（15 个都改）。

保存 → 重新构建 → 烧录 → 再用「全 A」keymap 按任意键测试。
